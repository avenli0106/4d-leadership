# 4D天性测评项目开发记录

## 项目概述

为用户开发一个基于 NASA 4D卓越团队系统的在线天性测评工具，最终部署为可分享的网页链接，并具备后台数据收集功能。

---

## 初始需求

用户原有微信保存的4D领导力测评链接失效，需要重新找一个或做一个。核心要求：
- 基于 NASA 4D 卓越团队天性测评（查理·佩勒林博士）
- 维度：直觉 vs 感觉，逻辑 vs 情感
- 四种颜色结果：绿色（培养）、黄色（包融）、蓝色（展望）、橙色（指导）
- 需要能发给朋友使用的在线链接
- 需要后台能看到谁填了、填了什么

---

## 技术方案演进

### 第一阶段：寻找现有链接
- 搜索了问卷星、微信文章、培训网站等渠道的现有测评链接
- 找到的有效链接均存在"只收数据不返结果"或"需邮件发送报告"的问题
- 决定自己开发一个完整的测评页面

### 第二阶段：自主开发测评页面

**核心逻辑**：
- 14道二选一题目（前7题测情感/逻辑，后7题测直觉/感觉）
- 计分规则：绿色=情感+直觉，黄色=情感+感觉，蓝色=逻辑+直觉，橙色=逻辑+感觉
- 结果展示：XY维度区域图（多边形面积直观显示四种颜色能量分布）

**部署方案**：
- GitHub Pages 静态托管（免费、稳定、可分享链接）
- Google Sheets + Apps Script 作为数据后台（免费、无需服务器）

## 问题排查记录

### 问题1：GitHub push 网络失败
**现象**：`fatal: unable to access 'https://github.com/...': Recv failure: Connection was reset`
**原因**：国内访问 GitHub HTTPS 不稳定
**解决**：配置 git 走本地代理（端口 18780），`git config http.proxy http://127.0.0.1:18780`

### 问题2：GitHub Pages 访问链接未立即出现
**原因**：Settings → Pages 保存后需刷新页面，部署有 1-2 分钟延迟
**解决**：刷新页面等待，链接格式为 `https://用户名.github.io/仓库名/`

### 问题3：测评后数据未写入 Google Sheets
**排查过程**：
1. GET 请求正常 → Web App URL 有效
2. 执行记录为空 → POST 请求未到达服务器
3. Network 面板显示 POST 404 → 部署版本仍是旧代码
**根因**：Google Apps Script 保存代码后未创建"新版本"部署，Web App 运行的还是旧版代码
**解决**：管理部署 → 修改 → 版本选「新版本」→ 重新部署

### 问题4：结果页显示不完整（只有区域图）
**根因**：`mainKey` 变量未传递到结果渲染函数，导致后续 DOM 渲染中断
**解决**：将 `mainKey` 加入全局结果数据对象中

## 技术栈

- **数据收集**：Google Apps Script + Google Sheets
- **字体**：系统默认字体栈（PingFang SC / Microsoft YaHei）

## 数据记录格式

2. 用户名
3. 情感(F) 得分
4. 直觉(N) 得分
5. 逻辑(T) 得分
6. 感觉(S) 得分
7. 绿色得分
8. 黄色得分
9. 蓝色得分
10. 橙色得分
11. 主导颜色
